# LyricsX 1.9.0-beta.6 What's New

This beta merges every change from the stable 1.8.3 release into the
Apple Music-style lyrics window development line.

## AppleMusic-style lyrics window

- **Native fullscreen + minimize button** — the window now supports
  macOS native fullscreen and exposes a minimize button.
- **Pin to top toggle** — keep the lyrics window always on top from
  the header.
- **Hover-aware control buttons** — control buttons surface on hover
  so they no longer crowd the lyrics while you're watching.
- **Karaoke fill across wrapped lines** — TextRenderer cascades the
  karaoke fill correctly across visual line wraps inside long lines.
- **Re-align highlighted line on resize** — ScrollView re-anchors the
  active line when the container size changes.
- **Rebalanced column spacing** — RootView column widths better
  accommodate longer titles.

## Performance & stability

- **Deduped currentTrackWillChange** — AppController suppresses a
  spurious lyrics re-search triggered by artwork updates on the same
  track.
- **Quieter widget logs** — AppController is now `final` and silences
  noisy widget timeline logs.

## Merged from 1.8.3

- macOS 10.15 (Catalina) compatibility restored. **Note: 1.8.3 is
  expected to be the final release branch that supports 10.15; the
  1.9.x line may drop it.**
- NowPlaying detection now identifies iOS-on-Mac apps as a source.
- Fixed "Launch with Player" no longer triggering — toggle the
  preference off and on once after upgrading to re-arm the helper.
- Main app dropped the sandbox entitlement. LyricsX is fully open
  source — audit the codebase at
  https://github.com/MxIris-LyricsX-Project/LyricsX.
