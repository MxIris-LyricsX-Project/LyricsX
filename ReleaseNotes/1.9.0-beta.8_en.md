# What's New

## New

- **Boost lyrics whose cover art matches the playing track** — A new switch under *Preferences → Lab* (off by default). When several providers return same-title results that differ mainly by their cover image, the right version often only stands out visually. With this on, each candidate's cover art is compared against the artwork of the track you're playing and visually matching results get a quality boost, so the correct version is more likely to win automatically. Matching uses a compact perceptual fingerprint (a luminance shape signature gated by average colour); cover downloads are throttled and cached per session, and the boost is kept out of the shared lyrics cache so it never affects unrelated lookups.
- **Source priority ranking** — A new *Enable source priority ranking* switch with a reorderable source list under *Preferences → Source*. When enabled, results from the sources you place higher are preferred during selection, letting you steer which provider wins when more than one returns usable lyrics.
- **Use source-provided kana for furigana** — A new switch under *Preferences → Lab*, paired with the existing furigana generation option. When a lyrics source already ships kana readings (for example QQMusic), the desktop karaoke overlay now uses those original readings for furigana instead of generating them, giving more accurate readings for Japanese lyrics.

## Improvements

- **Lighter, smoother Apple Music lyrics overlay** — The blurred backdrop behind the Apple Music lyrics window is now rendered once per track from a small thumbnail instead of re-blurring the full-size window image on every frame, cutting peak memory from hundreds of megabytes on large displays to a small fraction of that. The window and its refresh timer are also created only when you open it and fully released when you close it, so it no longer keeps running in the background for users who never open it.
