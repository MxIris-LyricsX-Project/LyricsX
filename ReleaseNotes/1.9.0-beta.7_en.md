# What's New

## New

- **Recover original song and artist names via MusicKit (Apple Music only)** — A new switch under *Preferences → Lab* (off by default). When Apple Music shows song or artist names in a localized form, enabling this switch queries MusicKit for the track's original, non-localized names and uses those to search for lyrics — a significant hit-rate improvement when searching for English songs from a non-English region. Turning it on requires the system "Apple Music" access authorization: the system prompt is shown at the moment you check it; if access is denied or restricted, the toggle snaps back off and points to *System Settings → Privacy & Security → Media & Apple Music*. The feature only takes effect when the currently selected player is Apple Music. Requires macOS 12.0+.
- **Strip brackets from the title before lyrics search (on by default)** — A new switch under *Preferences → General*. Bracket annotations such as `(feat. X)`, `[Explicit]`, or `【Live】` frequently cause zero-hit searches at providers. When enabled, both half-width and full-width `() [] （） 【】 ［］` are stripped from the title before the search request is built, while the full title still drives local cache lookup and lyrics metadata association. When *Strict Search* is also on, the bracket-stripped title is what the similarity check uses, so legitimate "Song (Remix)" results from providers still pass through.
